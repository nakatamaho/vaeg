# M2 — VS2017 / toolset v141, legacy code as-is

## Goal
The same unmodified sources build under VS2017 with platform toolset
v141, alongside (not replacing) the VS2008 project.

## Steps
1. Create `*.vcxproj` (+ `.sln`) by upgrading a COPY of the M1 project.
   Keep the VS2008 `.vcproj`/`.sln` untouched and buildable — both
   toolchains must pass every gate from G2 through G5.
2. In the v141 project set, explicitly:
   - `PlatformToolset` = v141; Win32 / Release primary target.
   - Charset flags, so the build does not depend on the host locale:
     `/source-charset:.932 /execution-charset:.932`
     (sources are still CP932 until M6; this silences C4819 on
     non-Japanese hosts and pins literal bytes).
   - `_MBCS`, matching M1.
   - Conformance relaxations as needed for legacy code
     (`/Zc:strictStrings-`, `/permissive` i.e. permissive mode ON) —
     record every relaxation in `docs/agents/reports/m2_deviations.md`.
3. Build. Same minimal-fix discipline as M1: prefer project-level
   settings over source edits; every source edit documented with
   file/line/reason. Do NOT start fixing the (large) new warning set;
   dump it to `docs/agents/reports/m2_warnings.txt` for later.
4. Verify the VS2008 build still passes after any shared-source edits.

## Machine checks
- v141 build log, 0 errors; VS2008 build log, 0 errors.
- `git diff --stat`: new project files + documented deviations only.

## GATE G2 (human)
User runs the G-checklist on the v141 binary (V3 mode, VA demo, OS).
On pass: tag `baseline-v141`.
